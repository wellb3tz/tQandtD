## 🎲 3D Volumetric Noise

### Описание
3D Simplex noise добавляет вертикальное измерение к генерации ландшафта, создавая более органичные и реалистичные формы рельефа.

### Параметры
- **Enable 3D Noise** - включить/выключить 3D генерацию
- **Z-Scale** (0.1 - 2.0) - масштаб по оси Z

### Технические детали
```javascript
noise3DConfig: {
  enable3D: true,
  zScale: 0.5  // Меньше = более плавные переходы
}
```

### Эффекты
- **Без 3D Noise**: Плоские, предсказуемые формы
- **С 3D Noise**: Объёмные, органичные формы с вертикальными вариациями

### Производительность
- Добавляет ~5-10ms к времени генерации чанка
- Рекомендуется для финальной визуализации

### Когда использовать
- ✅ Для реалистичных горных массивов
- ✅ Для пещер и подземных структур (будущая функция)
- ✅ Для органичных форм рельефа
- ❌ Для простых, плоских миров
- ❌ На слабых устройствах

### Smooth Transitions (Плавные переходы)

#### Параметры
- **Enable Transitions** - включить плавные переходы
- **Transition Width** (5-20) - ширина зоны перехода в тайлах

#### Как работает
```javascript
enhancedBiomeConfig: {
  enableTransitions: true,
  transitionWidth: 10  // 10 тайлов переходной зоны
}
```

В переходных зонах:
- Цвета плавно смешиваются между биомами
- Характеристики интерполируются
- Создаётся естественный градиент

#### Визуальный эффект
- **Без переходов**: Резкие границы между биомами
- **С переходами**: Плавные, естественные границы

## 📐 LOD System (Level of Detail)

### Описание
Система уровней детализации автоматически снижает качество дальних чанков для улучшения производительности.

### 3 Уровня детализации

#### 🔴 HIGH (Высокий)
- **Расстояние**: 0-2 чанка от игрока
- **Разрешение меша**: 100% (31×31 вершин)
- **Плотность объектов**: 100%
- **Все ресурсы**: Да
- **Все структуры**: Да
- **Производительность**: Нормальная

#### 🟡 MEDIUM (Средний)
- **Расстояние**: 2-5 чанков от игрока
- **Разрешение меша**: 50% (15×15 вершин)
- **Плотность объектов**: 50%
- **Ресурсы**: Только крупные кластеры
- **Структуры**: Только крупные
- **Производительность**: +30% FPS

#### 🟢 LOW (Низкий)
- **Расстояние**: 5+ чанков от игрока
- **Разрешение меша**: 25% (7×7 вершин)
- **Плотность объектов**: 10%
- **Ресурсы**: Нет
- **Структуры**: Только башни
- **Производительность**: +60% FPS

### Параметры
```javascript
lodConfig: {
  distances: [2, 5],                    // Пороги в чанках
  meshResolutions: [1.0, 0.5, 0.25],    // Множители разрешения
  featureDensities: [1.0, 0.5, 0.1]     // Множители плотности
}
```

### Когда использовать
- ✅ При большой дистанции рендеринга (>5 чанков)
- ✅ На слабых устройствах
- ✅ Для оптимизации производительности
- ❌ При малой дистанции рендеринга (<3 чанков)
- ❌ Для скриншотов высокого качества

## ⚡ Incremental Generation

### Описание
Поэтапная генерация чанков с временными бюджетами для поддержания 60fps.

### 5 Стадий генерации

#### 1️⃣ TERRAIN (Ландшафт)
- **Время**: ~30% от общего
- **Операции**: 
  - Генерация heightmap
  - Применение 3D noise (если включён)
  - Domain warping
- **Доступные данные**: heightmap

#### 2️⃣ BIOMES (Биомы)
- **Время**: ~20% от общего
- **Операции**:
  - Классификация биомов
  - Применение переходов
  - Генерация микро-биомов
  - Высотные пояса
- **Доступные данные**: heightmap, biomeMap, enhancedBiomeData

#### 3️⃣ RIVERS (Реки)
- **Время**: ~15% от общего
- **Операции**:
  - Поиск источников
  - Трассировка потоков
  - Создание речных сегментов
- **Доступные данные**: heightmap, biomeMap, rivers

#### 4️⃣ RESOURCES (Ресурсы)
- **Время**: ~20% от общего
- **Операции**:
  - Генерация кластеров
  - Фильтрация по биомам
  - Расчёт количества
- **Доступные данные**: heightmap, biomeMap, rivers, resources

#### 5️⃣ STRUCTURES (Структуры)
- **Время**: ~15% от общего
- **Операции**:
  - Poisson Disk Sampling
  - Проверка правил размещения
  - Создание структур
- **Доступные данные**: Все (чанк завершён)

### Параметры
```javascript
incrementalConfig: {
  enabled: true,
  timeBudgetMs: 16  // 16ms = 60fps, 33ms = 30fps
}
```

### Time Budget (Временной бюджет)

| Бюджет | FPS | Использование |
|--------|-----|---------------|
| 8ms    | 120fps | Высокопроизводительные системы |
| 16ms   | 60fps | Рекомендуется для большинства |
| 33ms   | 30fps | Слабые устройства |
| 50ms   | 20fps | Очень слабые устройства |

### Пример использования
```javascript
// Начать генерацию
const partial = chunkManager.getChunkIncremental(0, 0);

// Игровой цикл
function gameLoop() {
  // Продолжить генерацию
  const complete = chunkManager.continueGeneration(0, 0);
  
  if (!complete) {
    // Отобразить прогресс
    const partial = chunkManager.getChunkIncremental(0, 0);
    updateProgressBar(partial.stage, partial.progress);
    
    // Отобразить частичные данные
    if (partial.stage >= GenerationStage.TERRAIN) {
      renderTerrain(partial.data.heightmap);
    }
  } else {
    // Генерация завершена
    const chunk = chunkManager.getChunk(0, 0);
    renderFullChunk(chunk);
  }
  
  requestAnimationFrame(gameLoop);
}
```

## 🎨 Display Modes (Режимы отображения)

### 1. Biomes (Биомы)
**По умолчанию**

Отображает цвета биомов:
- Ocean: Тёмно-синий
- Beach: Светло-жёлтый
- Desert: Золотой
- Plains: Светло-зелёный
- Forest: Тёмно-зелёный
- Taiga: Изумрудный
- Tundra: Голубой
- Mountain: Серый

**Использование**: Общий обзор мира, навигация

### 2. Height Map (Карта высот)
Отображает высоту через цветовой градиент:
- Низкие области: Синий/фиолетовый
- Средние области: Зелёный/жёлтый
- Высокие области: Оранжевый/красный

Отображает приблизительную влажность:
- Сухие области (пустыни): Красный/оранжевый
- Умеренные области: Жёлтый/зелёный
- Влажные области (океаны, леса): Синий/голубой

**Использование**: Анализ климата, планирование размещения

### 4. Temperature (Температура)
Отображает приблизительную температуру:
- Холодные области (тундра, горы): Синий/фиолетовый
- Умеренные области: Зелёный
- Тёплые области (пустыни): Красный/оранжевый

**Использование**: Анализ климатических зон

{ 
  type: 'slope',

```javascript
{ 
  type: 'elevation', 
  params: { minHeight: 0.6 } 
}
```
Минимальная высота для размещения (0.0 - 1.0).

## 💾 World Serialization

### JSON Format
**Преимущества**:
- ✅ Читаемый формат
- ✅ Легко редактировать
- ✅ Совместимость с инструментами

**Недостатки**:
- ❌ Больший размер файла
- ❌ Медленнее загрузка

**Размер**: ~50KB на чанк (без компрессии)

### Binary Format
**Преимущества**:
- ✅ Компактный размер
- ✅ Быстрая загрузка
- ✅ Эффективная компрессия

**Недостатки**:
- ❌ Не читаемый формат
- ❌ Сложнее редактировать

**Размер**: ~15KB на чанк (с компрессией)

### Compression (Компрессия)
Использует pako (zlib):
- Уменьшает размер на 60-70%
- Добавляет ~5ms к времени сохранения
- Рекомендуется для больших миров

### Modification Tracking
Отслеживает изменения:
- Изменённые тайлы
- Добавленные структуры
- Удалённые структуры
- Изменения высоты

## 📊 Performance Statistics

### Метрики

#### FPS (Frames Per Second)
- **Цель**: 60 FPS
- **Минимум**: 30 FPS
- **Факторы**:
  - Render Distance
  - LOD включён/выключен
  - Количество объектов

#### Chunks Loaded
- **Оптимально**: 25-50 чанков
- **Максимум**: 100 чанков (ограничение кеша)
- **Формула**: (2 × RenderDistance + 1)²

#### Cache Size
- **По умолчанию**: 100 чанков
- **Память**: ~4.6MB (100 чанков × 46KB)
- **LRU**: Автоматическая выгрузка старых чанков

#### Gen Time (Generation Time)
- **Цель**: <100ms
- **Типично**: 20-50ms
- **Факторы**:
  - 3D Noise включён/выключен
  - Количество ресурсов
  - Количество структур

## 🎮 Best Practices

### Для разработчиков

1. **Начни с базовой конфигурации**
   - Отключи все продвинутые функции
   - Протестируй базовую генерацию
   - Постепенно добавляй функции

2. **Оптимизируй производительность**
   - Используй LOD для больших дистанций
   - Включи Incremental Generation
   - Настрой Time Budget под целевой FPS

3. **Тестируй на разных устройствах**
   - Слабые ПК: LOD + Incremental
   - Средние ПК: Все функции с оптимизацией
   - Мощные ПК: Все функции без ограничений

4. **Используй правильный формат сохранения**
   - JSON для разработки и отладки
   - Binary для продакшена

### Для игроков

1. **Настрой под свой ПК**
   - Начни с Render Distance = 3
   - Постепенно увеличивай до комфортного FPS
   - Включи LOD если FPS < 30

2. **Экспериментируй с seed**
   - Разные seed = разные миры
   - Запиши интересные seed
   - Поделись с друзьями

3. **Исследуй биомы**
