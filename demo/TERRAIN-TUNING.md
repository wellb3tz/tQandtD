# Terrain Tuning Guide - Настройка ландшафта

Руководство по настройке параметров генерации ландшафта для достижения желаемого результата.

## 🎯 Текущие настройки (оптимизированные)

Демо использует оптимизированные параметры для плавного, реалистичного рельефа:

```javascript
const HEIGHT_SCALE = 8;  // Множитель высоты в 3D пространстве

terrainConfig: {
  baseScale: 0.025,      // Масштаб шума (больше = крупнее формы)
  octaves: 3,            // Количество слоёв шума (меньше = плавнее)
  persistence: 0.45,     // Уменьшение амплитуды каждого слоя
  lacunarity: 2.0,       // Увеличение частоты каждого слоя
  warpStrength: 12,      // Сила искажения пространства
  heightMultiplier: 1.0  // Финальный множитель высоты
}
```

## 📊 Влияние параметров

### HEIGHT_SCALE (Множитель высоты в 3D)

Контролирует визуальную высоту рельефа в Three.js сцене.

| Значение | Эффект | Использование |
|----------|--------|---------------|
| 3-5 | Очень плоский | Равнины, пустыни |
| 6-10 | Плавный (рекомендуется) | Реалистичный ландшафт |
| 11-15 | Холмистый | Умеренные горы |
| 16-25 | Резкий | Экстремальные горы |
| 26+ | Очень резкий | Фантастические миры |

**Текущее**: 8 (плавный рельеф)

### baseScale (Масштаб шума)

Контролирует размер основных форм рельефа.

| Значение | Эффект | Описание |
|----------|--------|----------|
| 0.005-0.01 | Мелкие детали | Много мелких холмов |
| 0.015-0.025 | Средние формы (рекомендуется) | Естественные холмы и долины |
| 0.03-0.05 | Крупные формы | Большие горные массивы |
| 0.06+ | Очень крупные | Континентальный масштаб |

**Текущее**: 0.025 (средние формы)

**Правило**: Больше значение = крупнее и плавнее формы

### octaves (Количество слоёв шума)

Контролирует детализацию рельефа.

| Значение | Эффект | Производительность |
|----------|--------|--------------------|
| 1 | Очень гладко | Отлично |
| 2 | Гладко | Отлично |
| 3 | Плавно (рекомендуется) | Хорошо |
| 4 | Детально | Средне |
| 5-6 | Очень детально | Плохо |
| 7+ | Избыточно | Очень плохо |

**Текущее**: 3 (плавный рельеф)

**Правило**: Меньше octaves = плавнее рельеф, выше FPS

### persistence (Уменьшение амплитуды)

Контролирует вклад каждого следующего слоя шума.

| Значение | Эффект | Описание |
|----------|--------|----------|
| 0.2-0.3 | Очень гладко | Доминирует базовый слой |
| 0.4-0.5 | Плавно (рекомендуется) | Сбалансированные слои |
| 0.6-0.7 | Детально | Заметные мелкие детали |
| 0.8+ | Шумно | Слишком много деталей |

**Текущее**: 0.45 (плавный баланс)

**Правило**: Меньше значение = плавнее рельеф

### lacunarity (Увеличение частоты)

Контролирует как быстро увеличивается частота каждого слоя.

| Значение | Эффект | Описание |
|----------|--------|----------|
| 1.5 | Мягкие переходы | Плавные детали |
| 2.0 | Стандарт (рекомендуется) | Естественные детали |
| 2.5-3.0 | Резкие переходы | Контрастные детали |
| 3.5+ | Очень резкие | Искусственный вид |

**Текущее**: 2.0 (стандарт)

**Правило**: Обычно оставляют 2.0

### warpStrength (Сила искажения)

Контролирует domain warping - искажение пространства для органичных форм.

| Значение | Эффект | Описание |
|----------|--------|----------|
| 0 | Нет искажения | Правильные формы |
| 5-15 | Лёгкое (рекомендуется) | Естественные изгибы |
| 16-30 | Среднее | Заметные искажения |
| 31-50 | Сильное | Очень органичные формы |
| 51+ | Экстремальное | Хаотичные формы |

**Текущее**: 12 (лёгкое искажение)

**Правило**: Меньше значение = более правильные формы

## 🔧 Пошаговая настройка

### Проблема: Слишком резкий рельеф

**Решение**:
1. ⬇️ Уменьши `HEIGHT_SCALE` (например, с 20 до 8)
2. ⬆️ Увеличь `baseScale` (например, с 0.01 до 0.025)
3. ⬇️ Уменьши `octaves` (например, с 4 до 3)
4. ⬇️ Уменьши `persistence` (например, с 0.5 до 0.45)
5. ⬇️ Уменьши `warpStrength` (например, с 30 до 12)

### Проблема: Слишком плоский рельеф

**Решение**:
1. ⬆️ Увеличь `HEIGHT_SCALE` (например, с 8 до 12)
2. ⬇️ Уменьши `baseScale` (например, с 0.025 до 0.018)
3. ⬆️ Увеличь `octaves` (например, с 3 до 4)
4. ⬆️ Увеличь `persistence` (например, с 0.45 до 0.52)

### Проблема: Слишком мелкие детали

**Решение**:
1. ⬆️ Увеличь `baseScale` (например, с 0.025 до 0.035)
2. ⬇️ Уменьши `octaves` (например, с 3 до 2)
3. ⬇️ Уменьши `persistence` (например, с 0.45 до 0.4)

### Проблема: Слишком правильные формы

**Решение**:
1. ⬆️ Увеличь `warpStrength` (например, с 12 до 20)

### Проблема: Слишком хаотичные формы

**Решение**:
1. ⬇️ Уменьши `warpStrength` (например, с 30 до 12)
2. ⬇️ Уменьши `octaves` (например, с 4 до 3)

## 📐 Математика параметров

### Fractional Brownian Motion (fBm)

Движок использует fBm для генерации многослойного шума:

```
height = 0
amplitude = 1.0
frequency = baseScale

for i in 0..octaves:
  height += noise(x * frequency, y * frequency) * amplitude
  amplitude *= persistence
  frequency *= lacunarity
```

### Влияние параметров на формулу

- **baseScale**: Начальная частота (масштаб первого слоя)
- **octaves**: Количество итераций цикла
- **persistence**: Множитель амплитуды (обычно 0.3-0.7)
- **lacunarity**: Множитель частоты (обычно 2.0)

### Примеры расчёта

**Конфигурация 1** (плавная):
```
baseScale = 0.025, octaves = 3, persistence = 0.45

Слой 1: freq = 0.025, amp = 1.0
Слой 2: freq = 0.05,  amp = 0.45
Слой 3: freq = 0.1,   amp = 0.2025

Итого: 3 слоя, макс. амплитуда = 1.6525
```

**Конфигурация 2** (детальная):
```
baseScale = 0.01, octaves = 4, persistence = 0.5

Слой 1: freq = 0.01, amp = 1.0
Слой 2: freq = 0.02, amp = 0.5
Слой 3: freq = 0.04, amp = 0.25
Слой 4: freq = 0.08, amp = 0.125

Итого: 4 слоя, макс. амплитуда = 1.875
```

## 💡 Советы по оптимизации

### Производительность vs Качество

| Приоритет | octaves | baseScale | warpStrength |
|-----------|---------|-----------|--------------|
| Максимальная производительность | 2 | 0.03 | 5 |
| Сбалансированный (рекомендуется) | 3 | 0.025 | 12 |
| Максимальное качество | 4-5 | 0.015 | 20 |

### Влияние на время генерации

- **octaves**: +25% времени на каждый слой
- **warpStrength**: +10% времени при значении >20
- **baseScale**: Минимальное влияние

### Рекомендации

1. **Начни с octaves = 3** - оптимальный баланс
2. **Настрой HEIGHT_SCALE** - самый простой способ изменить вид
3. **Подбери baseScale** - для нужного размера форм
4. **Тонкая настройка persistence** - для детализации
5. **warpStrength в последнюю очередь** - для органичности
