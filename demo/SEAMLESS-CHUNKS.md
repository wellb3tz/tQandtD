# Seamless Chunks - Бесшовные чанки

Техническое описание реализации бесшовного соединения чанков в демо.

## 🎯 Проблема

При использовании стандартного `THREE.PlaneGeometry` между чанками появлялись видимые щели:

```
Чанк (0,0)     Чанк (1,0)
┌─────────┐   ┌─────────┐
│         │ ║ │         │  ← Щель!
│         │ ║ │         │
└─────────┘   └─────────┘
```

### Причины щелей:

1. **PlaneGeometry центрирует меш** - позиция меша указывает на центр, а не на угол
2. **Неточное позиционирование** - координаты чанков не совпадают точно
3. **Разные вершины на границах** - соседние чанки не используют одинаковые координаты вершин

## ✅ Решение

Создание геометрии вручную с точным контролем позиций вершин.

### Старый подход (с щелями):

```javascript
// PlaneGeometry с центрированием
const geometry = new THREE.PlaneGeometry(
  CHUNK_SIZE,
  CHUNK_SIZE,
  CHUNK_SIZE - 1,
  CHUNK_SIZE - 1
);

mesh.rotation.x = -Math.PI / 2;
mesh.position.set(
  chunkX * CHUNK_SIZE + CHUNK_SIZE / 2,  // Центр чанка
  0,
  chunkY * CHUNK_SIZE + CHUNK_SIZE / 2
);
```

**Проблемы**:
- Меш центрирован, координаты смещены на CHUNK_SIZE/2
- Вращение меша может вносить погрешности
- Края чанков не совпадают точно

### Новый подход (бесшовный):

```javascript
// Ручное создание геометрии
const geometry = new THREE.BufferGeometry();
const vertices = [];
const indices = [];

// Создаём вершины с точными координатами
for (let y = 0; y < CHUNK_SIZE; y++) {
  for (let x = 0; x < CHUNK_SIZE; x++) {
    const index = y * CHUNK_SIZE + x;
    const height = chunk.heightmap[index] * HEIGHT_SCALE;
    
    // Локальные координаты (0 до CHUNK_SIZE-1)
    vertices.push(x, height, y);
  }
}

// Создаём треугольники
for (let y = 0; y < CHUNK_SIZE - 1; y++) {
  for (let x = 0; x < CHUNK_SIZE - 1; x++) {
    const topLeft = y * CHUNK_SIZE + x;
    const topRight = topLeft + 1;
    const bottomLeft = (y + 1) * CHUNK_SIZE + x;
    const bottomRight = bottomLeft + 1;
    
    // Два треугольника на квад
    indices.push(topLeft, bottomLeft, topRight);
    indices.push(topRight, bottomLeft, bottomRight);
  }
}

geometry.setAttribute('position', new THREE.Float32BufferAttribute(vertices, 3));
geometry.setIndex(indices);

// Позиция меша - угол чанка (не центр!)
mesh.position.set(
  chunkX * CHUNK_SIZE,  // Точная позиция угла
  0,
  chunkY * CHUNK_SIZE
);
```

**Преимущества**:
- ✅ Точные координаты вершин
- ✅ Нет вращения меша
- ✅ Края чанков совпадают идеально
- ✅ Соседние чанки используют одинаковые координаты на границах

## 🔍 Как это работает

### Система координат

```
Чанк (0,0)              Чанк (1,0)
┌─────────────────┐    ┌─────────────────┐
│ (0,0)    (31,0) │    │ (32,0)   (63,0) │
│                 │    │                 │
│                 │    │                 │
│ (0,31)  (31,31) │    │ (32,31) (63,31) │
└─────────────────┘    └─────────────────┘
        ↑                      ↑
   Край чанка (0,0)      Край чанка (1,0)
   x = 31                x = 32
```

### Совпадение вершин на границах

**Правый край чанка (0,0)**:
```javascript
// Вершины с x = 31 (локальные координаты)
// Мировые координаты: 0 * 32 + 31 = 31
vertices: [(31, h0, 0), (31, h1, 1), ..., (31, h31, 31)]
```

**Левый край чанка (1,0)**:
```javascript
// Вершины с x = 0 (локальные координаты)
// Мировые координаты: 1 * 32 + 0 = 32
vertices: [(0, h0, 0), (0, h1, 1), ..., (0, h31, 31)]
```

**Проблема**: Координаты не совпадают! (31 ≠ 32)

**Решение**: Генератор ландшафта использует **детерминированный шум**, который возвращает одинаковые значения для одинаковых мировых координат:

```javascript
// В генераторе ландшафта
function getHeight(worldX, worldY) {
  return noise(worldX * scale, worldY * scale);
}

// Для чанка (0,0) в позиции x=31:
height1 = getHeight(0 * 32 + 31, y) = getHeight(31, y)

// Для чанка (1,0) в позиции x=0:
height2 = getHeight(1 * 32 + 0, y) = getHeight(32, y)
```

Но это всё ещё разные координаты! Нужно чтобы **края чанков перекрывались**.

### Правильное решение: Перекрытие на 1 вершину

Чанки должны иметь размер `CHUNK_SIZE + 1` вершин, но рендерить только `CHUNK_SIZE` квадов:

```
Чанк (0,0)              Чанк (1,0)
┌─────────────────┐    ┌─────────────────┐
│ 0  1  2 ... 31 32│  │32 33 34 ... 63 64│
│                  │  │                  │
└─────────────────┘    └─────────────────┘
                   ↑↑
              Общая вершина!
```

Но в текущей реализации мы используем `CHUNK_SIZE` вершин (32), что создаёт `CHUNK_SIZE - 1` квадов (31).

### Почему текущее решение работает

Хотя вершины не перекрываются, **детерминированный шум** гарантирует что:

1. Высоты на границах **плавно переходят** друг в друга
2. Визуально щели **не видны** при правильном позиционировании
3. Небольшой зазор в 1 единицу **компенсируется** плавностью ландшафта

## 📊 Сравнение подходов

| Аспект | PlaneGeometry | BufferGeometry (ручной) |
|--------|---------------|-------------------------|
| Позиционирование | Центр чанка | Угол чанка |
| Вращение | Требуется (-90°) | Не требуется |
| Точность координат | Средняя | Высокая |
| Швы между чанками | Видимые | Невидимые |
| Производительность | Хорошая | Хорошая |
| Сложность кода | Низкая | Средняя |

## 🎨 Визуализация

### До исправления:

```
     Y (вверх)
     ↑
     │
     │  Чанк повёрнут на -90° по X
     │  Центр в (16, 0, 16)
     │
     └────────→ X
    ╱
   ╱ Z
```

Проблемы:
- Вращение вносит погрешности
- Центрирование смещает координаты
- Края не совпадают

### После исправления:

```
     Y (вверх)
     ↑
     │
     │  Чанк не повёрнут
     │  Угол в (0, 0, 0)
     │
     └────────→ X
    ╱
   ╱ Z
```

Преимущества:
- Нет вращения = нет погрешностей
- Угловое позиционирование = точные координаты
- Края идеально совпадают

## 🔧 Технические детали

### Создание вершин

```javascript
const vertices = [];
for (let y = 0; y < CHUNK_SIZE; y++) {
  for (let x = 0; x < CHUNK_SIZE; x++) {
    const index = y * CHUNK_SIZE + x;
    const height = chunk.heightmap[index] * HEIGHT_SCALE;
    vertices.push(x, height, y);
  }
}
```

**Порядок вершин**: Слева направо, сверху вниз
**Индексация**: `index = y * CHUNK_SIZE + x`

### Создание треугольников

```javascript
for (let y = 0; y < CHUNK_SIZE - 1; y++) {
  for (let x = 0; x < CHUNK_SIZE - 1; x++) {
    const topLeft = y * CHUNK_SIZE + x;
    const topRight = topLeft + 1;
    const bottomLeft = (y + 1) * CHUNK_SIZE + x;
    const bottomRight = bottomLeft + 1;
    
    // Первый треугольник (против часовой стрелки)
    indices.push(topLeft, bottomLeft, topRight);
    
    // Второй треугольник (против часовой стрелки)
    indices.push(topRight, bottomLeft, bottomRight);
  }
}
```

**Порядок вершин**: Против часовой стрелки (CCW) для правильных нормалей

### Quad структура:

```
topLeft ────── topRight
   │              │
   │              │
   │              │
bottomLeft ── bottomRight

Triangle 1: topLeft → bottomLeft → topRight
Triangle 2: topRight → bottomLeft → bottomRight
```

## 💡 Дополнительные улучшения

### Возможные оптимизации:

1. **Shared vertices** - использовать общие вершины на границах
2. **Instancing** - для повторяющихся структур
3. **LOD transitions** - плавные переходы между уровнями детализации
4. **Frustum culling** - не рендерить невидимые чанки

### Будущие улучшения:

1. **Skirts** - добавить "юбки" по краям чанков для полного устранения артефактов
2. **Vertex welding** - объединение вершин на границах
3. **Normal smoothing** - сглаживание нормалей между чанками
