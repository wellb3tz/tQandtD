# Руководство по ручному тестированию LOD системы

## Цель
Проверить исправления производительности и LOD системы в реальном приложении.

## Подготовка

### 1. Запуск demo приложения

```bash
# Установить зависимости (если еще не установлены)
npm install

# Запустить dev сервер
npm run dev:demo
```

Приложение должно открыться в браузере по адресу `http://localhost:5173` (или другому порту, указанному в консоли).

### 2. Открыть инструменты разработчика

- **Chrome/Edge**: `F12` или `Ctrl+Shift+I` (Windows/Linux) / `Cmd+Option+I` (Mac)
- **Firefox**: `F12` или `Ctrl+Shift+I` (Windows/Linux) / `Cmd+Option+I` (Mac)

Перейдите на вкладку **Console** для мониторинга ошибок.

## Тестовые сценарии

### Сценарий 1: Базовая проверка LOD системы

**Шаги**:
1. В UI найдите секцию "LOD System" или "Performance"
2. Убедитесь, что LOD система **включена** (checkbox или toggle)
3. Нажмите кнопку генерации мира или загрузите существующий мир
4. Загрузите начальные чанки (обычно 9 чанков - 3x3 сетка)

**Ожидаемый результат**:
- ✅ Ландшафт отображается корректно
- ✅ Нет ошибок в консоли (особенно ошибок типа "Heightmap size mismatch")
- ✅ Нет визуальных артефактов (дыр, неправильной геометрии)

**Что проверять**:
- [ ] Ландшафт выглядит естественно
- [ ] Нет дыр или разрывов между чанками
- [ ] Консоль чистая (нет красных ошибок)

---

### Сценарий 2: Проверка производительности с большим количеством чанков

**Шаги**:
1. Убедитесь, что LOD система включена
2. Загрузите большое количество чанков:
   - Используйте кнопку "Load More Chunks" или аналогичную
   - Или двигайте камеру, чтобы загрузить новые чанки
   - Цель: загрузить **50+ чанков** (7x7 или больше)
3. Наблюдайте за FPS счетчиком в UI

**Ожидаемый результат**:
- ✅ FPS >= 60 при 9-25 чанках (3x3 - 5x5)
- ✅ FPS >= 45 при 49-100 чунках (7x7 - 10x10)
- ✅ Приложение остается отзывчивым
- ✅ Нет зависаний или лагов

**Что проверять**:
- [ ] FPS отображается в UI (если есть Performance Monitor)
- [ ] FPS стабилен и не падает ниже 45
- [ ] Камера двигается плавно
- [ ] Нет задержек при загрузке новых чанков

**Если FPS счетчика нет в UI**:
- Откройте консоль браузера
- Введите: `performance.now()` и запомните значение
- Подождите 1 секунду
- Снова введите: `performance.now()` 
- Разница должна быть ~1000ms (если нет лагов)

---

### Сценарий 3: Проверка LOD уровней

**Шаги**:
1. Загрузите мир с LOD системой
2. Двигайте камеру вверх (отдаляйтесь от ландшафта)
3. Наблюдайте за изменением детализации чанков

**Ожидаемый результат**:
- ✅ Ближние чанки имеют высокую детализацию (HIGH LOD)
- ✅ Средние чанки имеют среднюю детализацию (MEDIUM LOD)
- ✅ Дальние чанки имеют низкую детализацию (LOW LOD)
- ✅ Переходы между LOD уровнями плавные (без резких изменений)

**Что проверять**:
- [ ] Визуально видно, что дальние чанки менее детализированы
- [ ] Нет "попкорн эффекта" (резкого появления/исчезновения деталей)
- [ ] Ландшафт остается связным на всех расстояниях

**Визуальные признаки LOD уровней**:
- **HIGH LOD**: Много вершин, детальный рельеф, видны ресурсы и структуры
- **MEDIUM LOD**: Меньше вершин, упрощенный рельеф, видны структуры
- **LOW LOD**: Минимум вершин, сглаженный рельеф, нет ресурсов/структур

---

### Сценарий 4: Проверка Frustum Culling

**Шаги**:
1. Загрузите мир с большим количеством чанков (25+)
2. Поверните камеру так, чтобы часть чанков оказалась за спиной
3. Откройте консоль и проверьте статистику (если доступна)

**Ожидаемый результат**:
- ✅ Чанки вне поля зрения не рендерятся
- ✅ FPS увеличивается, когда меньше чанков в поле зрения
- ✅ Чанки плавно появляются/исчезают при повороте камеры

**Что проверять**:
- [ ] При повороте камеры на 180° FPS увеличивается
- [ ] Нет мерцания чанков при движении камеры
- [ ] Статистика показывает меньше видимых чанков при повороте

**Проверка статистики** (если доступна в UI):
- Visible Chunks: должно уменьшаться при повороте
- Hidden Chunks: должно увеличиваться при повороте
- Total Chunks: остается постоянным

---

### Сценарий 5: Проверка консоли на ошибки

**Шаги**:
1. Откройте консоль браузера (F12 → Console)
2. Очистите консоль (кнопка 🚫 или Ctrl+L)
3. Выполните все предыдущие сценарии
4. Проверьте консоль на наличие ошибок

**Ожидаемый результат**:
- ✅ Нет красных ошибок (errors)
- ✅ Допустимы информационные сообщения (info/log)
- ✅ Допустимы предупреждения (warnings), если они не критичны

**Критичные ошибки** (не должны появляться):
- ❌ "Heightmap size mismatch"
- ❌ "Cannot read property of undefined"
- ❌ "Index out of bounds"
- ❌ "WebGL context lost"
- ❌ Любые ошибки, связанные с LOD или heightmap

**Допустимые сообщения**:
- ℹ️ "LOD system enabled"
- ℹ️ "Chunk loaded: (x, y)"
- ⚠️ "Performance warning: FPS below 60" (если FPS действительно низкий)

---

### Сценарий 6: Стресс-тест

**Шаги**:
1. Загрузите максимальное количество чанков (100+)
2. Быстро двигайте камеру в разных направлениях
3. Включайте/выключайте различные слои (реки, ресурсы, структуры)
4. Наблюдайте за стабильностью приложения

**Ожидаемый результат**:
- ✅ Приложение не крашится
- ✅ FPS остается приемлемым (>= 30)
- ✅ Нет утечек памяти (проверить через DevTools → Memory)
- ✅ Камера остается отзывчивой

**Что проверять**:
- [ ] Приложение работает стабильно 5+ минут
- [ ] Нет зависаний или фризов
- [ ] Память не растет бесконечно (проверить через Task Manager)
- [ ] Можно продолжать взаимодействовать с UI

## Чек-лист финальной проверки

Перед завершением тестирования убедитесь:

### Функциональность
- [ ] LOD система включается/выключается через UI
- [ ] Ландшафт отображается корректно на всех LOD уровнях
- [ ] Нет визуальных артефактов (дыр, разрывов, неправильной геометрии)
- [ ] Ресурсы и структуры отображаются корректно
- [ ] Реки отображаются корректно

### Производительность
- [ ] FPS >= 60 при 9-25 чанках
- [ ] FPS >= 45 при 49-100 чунках
- [ ] Frustum culling работает (чанки скрываются вне поля зрения)
- [ ] LOD уровни применяются корректно (визуально)
- [ ] Нет лагов при движении камеры

### Стабильность
- [ ] Нет ошибок в консоли
- [ ] Приложение не крашится
- [ ] Нет утечек памяти
- [ ] Можно работать с приложением 5+ минут без проблем

### Пользовательский опыт
- [ ] Интерфейс отзывчивый
- [ ] Переходы между LOD уровнями плавные
- [ ] Загрузка чанков не вызывает фризов
- [ ] Общее впечатление улучшилось по сравнению с предыдущей версией

## Отчет о проблемах

Если вы обнаружили проблемы, запишите следующую информацию:

### Информация о проблеме
- **Описание**: Что пошло не так?
- **Шаги воспроизведения**: Как воспроизвести проблему?
- **Ожидаемое поведение**: Что должно было произойти?
- **Фактическое поведение**: Что произошло на самом деле?

### Технические детали
- **Браузер**: Chrome/Firefox/Safari + версия
- **ОС**: Windows/Mac/Linux
- **Разрешение экрана**: 1920x1080 и т.д.
- **Количество чанков**: Сколько чанков было загружено?
- **LOD уровень**: На каком LOD уровне произошла проблема?
- **FPS**: Какой был FPS в момент проблемы?

### Скриншоты и логи
- Сделайте скриншот проблемы
- Скопируйте ошибки из консоли
- Запишите видео, если проблема связана с анимацией/производительностью

## Дополнительные советы

### Оптимальные настройки для тестирования
- Используйте режим "Performance" в DevTools для профилирования
- Отключите расширения браузера, которые могут влиять на производительность
- Закройте другие вкладки для чистоты эксперимента

### Инструменты для измерения производительности
- **Chrome DevTools → Performance**: Записать профиль производительности
- **Chrome DevTools → Memory**: Проверить утечки памяти
- **Stats.js**: Если интегрирован в приложение, показывает FPS в реальном времени

### Горячие клавиши (если реализованы)
- `W/A/S/D`: Движение камеры
- `Q/E`: Вверх/вниз
- `Shift`: Ускорение движения
- `F`: Переключение полноэкранного режима
- `H`: Показать/скрыть помощь
